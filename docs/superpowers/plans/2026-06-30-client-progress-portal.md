# Client Progress Portal — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A `project_director`/`super_admin` issues a temporary, read-only LINE login that lets a project's client watch progress (summary · WP status · approved photos · report PDFs) for one project, with a valid-until date and early revoke.

**Architecture:** New `client` user role (ADR 0067). A `client_portal_access` row binds one client user → one project with `expires_at`/`revoked_at`; a `client_invites` row carries the single-use, 14-day claim token. The whole thing mirrors the proven spec-130/ADR-0051 contractor portal (claim-token + scoped-RLS external login) but with its own role, its own `/client` route, and dedicated read-only RLS arms — never widening a staff query.

**Tech Stack:** Next.js 16 App Router (Server Components), Supabase Postgres + RLS + `security definer` RPCs, LINE OAuth, pgTAP (`pnpm db:test`), Vitest, Playwright. Package manager **pnpm**.

**Spec:** `docs/feature-specs/233-client-progress-portal.md` (read it first).

## Global Constraints

- **TDD, test-first.** First message of every unit is the failing test. No production code before a failing test (CLAUDE.md).
- **Scope discipline.** Implement only what the spec says. No extra fields/helpers/handlers.
- **Every table has RLS.** New tables: `enable row level security` + `revoke all from anon, authenticated` + explicit grants.
- **Source definer RPC bodies from LIVE** (`pg_get_functiondef`), never from a migration file. Clone the _current_ `create_contractor_invite` / `claim_contractor_invite` — they hash tokens (migration `20260813024000`); the client RPCs must hash identically.
- **Enum value add goes in its own migration** (`ALTER TYPE … ADD VALUE` can't share a txn — ADR 0008 precedent).
- **Definer RPCs:** `revoke execute from public, anon;` `grant execute to authenticated;` null-safe role gate (RLS self-check coalesce trap — coalesce role check to false).
- **Schema is single-lane.** U1 + U2 touch `supabase/migrations/`. The lane is currently held by the ADR0066 chain. Do NOT start U1/U2 until the lane is free; claim it in `D:\claude\projects\prc-ops\LANES.md` with a migration timestamp = current-max + 1000 before pushing.
- **Danger-path = operator-held.** Every PR here touches migrations / `src/lib/auth/**` / RLS / new role → the autonomous-build fence HOLDS it for one-tap operator merge. Do not expect auto-merge.
- **No money, ever, in any client query.** Dedicated readers select only safe columns; column grants are the backstop.
- **Thai-first UI.** User-facing copy in Thai, matching existing portal tone. Reuse `src/lib/ui/classes` tokens (`CARD`, `BUTTON_PRIMARY`, etc.) — raw Tailwind palette is banned.
- **Issuer/revoker gate = `project_director` + `super_admin` only** (NOT `project_manager`). PM_ROLES already contains `project_director` AND `project_manager`, so a NEW dedicated set is required.
- **Per repo workflow:** one unit per session; update `docs/progress-tracker.md` at start/end; run `pnpm lint && pnpm typecheck && pnpm test` (+ `pnpm db:test` for schema units) before pushing; ship via `scripts/ship-pr.sh`.

```sql
-- Spec 233 / ADR 0067 — temporary scoped client portal access + claim tokens.

create table public.client_portal_access (
  id          uuid primary key default gen_random_uuid(),
  user_id     uuid not null references public.users(id),
  project_id  uuid not null references public.projects(id) on delete cascade,
  granted_by  uuid not null references public.users(id),
  granted_at  timestamptz not null default now(),
  expires_at  timestamptz,
  revoked_at  timestamptz,
  revoked_by  uuid references public.users(id),
  unique (user_id, project_id)
);
create index client_portal_access_project_idx on public.client_portal_access (project_id);

create table public.client_invites (
  id                uuid primary key default gen_random_uuid(),
  token_hash        text not null unique,          -- hashed, mirror contractor_invites (mig 20260813024000)
  project_id        uuid not null references public.projects(id) on delete cascade,
  access_expires_at timestamptz not null,
  created_by        uuid not null references public.users(id),
  created_at        timestamptz not null default now(),
  claimed_at        timestamptz,
  claimed_by        uuid references public.users(id)
);

alter table public.client_portal_access enable row level security;
alter table public.client_invites        enable row level security;
revoke all on public.client_portal_access from anon, authenticated;
revoke all on public.client_invites        from anon, authenticated;

-- Live-access predicate. SECURITY DEFINER so the client read arms can call it
-- without a recursive grant; body only reads its own auth.uid() rows.
create function public.client_has_live_access(p_project uuid)
returns boolean language sql stable security definer set search_path = public as $$
  select exists (
    select 1 from public.client_portal_access a
    where a.user_id = (select auth.uid())
      and a.project_id = p_project
      and a.revoked_at is null
      and (a.expires_at is null or a.expires_at > now())
  );
$$;
revoke execute on function public.client_has_live_access(uuid) from public, anon;
grant execute on function public.client_has_live_access(uuid) to authenticated;

-- Manage arms: PD + super only (mirrors CLIENT_ISSUER_ROLES).
grant select on public.client_portal_access to authenticated;
create policy "client access manage by director"
  on public.client_portal_access for all to authenticated
  using ((select public.current_user_role()) in ('project_director','super_admin'))
  with check ((select public.current_user_role()) in ('project_director','super_admin'));
-- The client reads only its own binding row.
create policy "client reads own access"
  on public.client_portal_access for select to authenticated
  using (user_id = (select auth.uid()));

-- client_invites: PD/super manage; no client/anon read (claim goes through the RPC).
grant select on public.client_invites to authenticated;
create policy "client invites manage by director"
  on public.client_invites for all to authenticated
  using ((select public.current_user_role()) in ('project_director','super_admin'))
  with check ((select public.current_user_role()) in ('project_director','super_admin'));

-- Dedicated client read arms — ADDITIONAL policies; never edit a staff arm.
-- Confirm each table's real project linkage before writing (photo_logs hangs off
-- work_packages; reports via params). Resolve the owning project, then gate on
-- client_has_live_access. Example (projects):
create policy "client reads own project"
  on public.projects for select to authenticated
  using ((select public.current_user_role()) = 'client'
         and public.client_has_live_access(id));
-- work_packages:
create policy "client reads project work_packages"
  on public.work_packages for select to authenticated
  using ((select public.current_user_role()) = 'client'
         and public.client_has_live_access(project_id));
-- photo_logs (approved only — confirm approval column/state; resolve project via WP):
-- reports (resolve project via the report's project linkage):
--   write both arms after confirming the columns from the live schema.
```
