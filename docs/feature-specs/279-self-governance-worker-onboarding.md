## 1. Problem & goal

`workers` is empty firm-wide (spec266u1 wiped it — 1 test row deleted), `labor_logs` = 0 all-time, and the real workforce (~30 named hands, 6 crews under a หัวหน้า, on TFM โพธิ์ทอง) exists only on a hand-written Daily Report. **Spec 278 U2's attendance muster reads "workers on this project" and returns nobody → dead on arrival.** The prerequisite is an **airtight self-governance onboarding flow**: each crew-lead onboards/manages their own crew, producing durable `workers` rows the muster groups by crew — while the gameable money-adjacent attributes (**level, pay-rate, pay-class, employment-tenure**) stay confirmed by a disinterested **PM/PD/super_admin** hand (ADR 0060 §5, so self-governance never becomes self-dealing), and every person's PII rests on a **defensible lawful basis** (ADR 0061 §4 / Thai PDPA).

**Scope:** the _roster_ (identity → crew → pay-ready → optional app-claim → lifecycle) + the _crew↔WP edge_. NOT the muster (278 U2), the attendance→labor_log conversion, or the WP even-split — but it **hard-wires the gates** those depend on.

## 2. The airtight staged flow (each gate blocks the next — no incomplete person leaks into cost/pay/muster)

**Stage 0 — Bootstrap** (one-time, top-down, per project). _Who:_ STAFF*APPROVAL_ROLES {procurement_manager, project_director, super_admin}. \_What:* per crew — create the lead as a **bound daily worker** (`create_worker`, back-office), issue `create_worker_invite` → LINE-claim sets `workers.user_id` + `role='contractor'`; create a `crews` row binding `lead_worker_id`. _Gate:_ an active-lead `crews` row exists. Until the lead claims, a back-office actor operates on the crew's behalf. **Leads never self-appoint** (H-BOOT).

**Stage 1 — IDENTITY.** _Who:_ crew-lead (own crew) **or** SA/PM proxy-operator (D1). _What:_ `full_name` (req), `phone` (opt), **Thai national-ID** (req for phoneless — dedup key, mod-11 checksum), `id_card` photo (deferred-uploadable), `date_of_birth` (req — age gate), tenure hint (advisory). Mints durable `PRC-YY-NNNN` (`employee_id_counters`). Writes a `crew_registrations` **staging** row (`pending`) — no `workers` row yet. _Gate (identity floor):_ name + valid national-ID + DOB + age≥18. National-ID collision firm-wide (active OR inactive) **hard-blocks** → offers move/reactivate, never a 2nd create (H-DEDUP/GHOST/REACT).

**Stage 2 — CONSENT / LAWFUL BASIS.** _Who:_ phone-owner (self) OR lead-attested **+ neutral witness** (SA/PM). _What:_ a lawful-basis record on the worker + a delivered versioned privacy-notice artifact. _Gate (basis floor):_ a live basis record of an accepted kind exists (see §5). Sensitive data minimised (H-SENS). **⚖️ counsel-pending** — see §5 / §9.

**Stage 3 — CLASSIFY + PAY-READY.** _Who:_ **PM/PD** sets `pay_type` + `day_rate`(pay) + `employment_type`; **super_admin** sets `level` (→ sell). _What:_ `approve_crew_registration` promotes staging → real `workers` row (**inlined INSERT**, not a `create_worker` call — H-NEST), copies PII+id*card, inserts `crew_members`, calls `assign_worker_to_project`. Member inherits `crews.default_day_rate` so a DC hand is **payable Friday** without waiting on per-head confirm (H-PAY) — `day_rate` never 0/NULL. \_Gate (cost-loggable → `cost_confirmed_at` stamped):* `level` + `day_rate` + `pay_type` + `employment_type` all set. Until stamped: rostered, attendable, payable-at-default, but **excluded from every labor_log** (the single choke-point gate — H-ZERO/H-LEVELNULL) so ungraded/unpriced labor never enters the cost engine.

**Stage 4 — APP-CLAIM** (optional, phone-owners). _Who:_ member self-claims; invite issued by **back-office** (NOT the lead — H-INVITE), post-confirmation. On claim: bind `user_id`, `role='contractor'`, prompt a **subject-given** consent affirmation (`record_worker_consent('pdpa_data')`). _Gate:_ invite refuses a still-pending worker.

**Stage 5 — DEACTIVATE / LIFECYCLE.** Lead retires OWN members; **reactivate = PM+ only**; **re-create of a deactivated person hard-blocked** (national-ID, H-REACT) → routed to PM reactivation; `do_not_rehire` refuses for-cause re-onboard. `deactivated_at` + reason (spec 235, revived); cascade move-out. Retiring a worker who is any crew's `lead_worker_id` **RAISES** unless the crew is first reassigned/disbanded (H-ORPHAN).

## 3. Roles & self-governance model

**Crew-lead = a bound WORKER, not an app-role** (verified: `claim_worker_invite` forces `role='contractor'`, so a "site_owner crew-lead" is impossible against the live claim RPC). Authority = the predicate `current_user_worker_id() = crews.lead_worker_id` (own-crew), never `current_user_role()`. A `role='contractor'` user must be routed to the crew UI (role-home change, Unit 4).

**Proxy-operator** (adoption reality — a tired phoneless หัวหน้า won't type; อนัญญา already authors the paper roster): `crews` carries `lead_worker_id` (accountable head, for cost attribution) **and** an optional `operator_user_id` (SA/PM who may enter data for this crew). Write gate = `current_user_worker_id() = lead_worker_id OR auth.uid() ∈ operator set`. Onboarding is a **one-time evening setup**, OFF the 8am morning-talk critical path.

**Money-governance split** (anti-gaming, ADR 0060 §5) — enforced by **parameter absence** (the lead RPC has NO money params; pinned by a `pg_proc.proargnames` test):

| Attribute                                                        | Who writes                                                          | Why                                                                      |
| ---------------------------------------------------------------- | ------------------------------------------------------------------- | ------------------------------------------------------------------------ |
| name, phone, national-ID, DOB, id_card, crew, project, WP-bundle | **lead / proxy**                                                    | operational, no P&L effect                                               |
| `day_rate` (pay/cost)                                            | PM/PD (`set_worker_day_rate`) + crew `default_day_rate` inheritance | per-head DC cost (hidden from lead)                                      |
| `pay_type`                                                       | PM/PD (`approve_crew_registration`)                                 | cost bucket (payroll vs DC)                                              |
| `employment_type`                                                | **PM/PD** (NOT lead — H-EMPTYPE)                                    | LIVE coin-weight + external-sell + vesting lever                         |
| `level`                                                          | **super_admin** (`set_worker_level`)                                | **sell-rate key** (the 560 the lead sees) + coin weight; anti-favoritism |

**Money visibility (D4):** lead (WP owner) sees the **sell/560 = WP cost** for his crew; pay/500 + markup/60 hidden; suppressed under SA-proxy view.

**RLS arms:** lead reads own crew via a DEFINER `get_my_crew_members()` (hand-selected safe cols — **excludes `level`, `note`, all money**; RLS is row- not column-level, H-COLSCOPE). `crew_registrations`/attachments = **service-role-only**, surfaced via DEFINER RPCs scoped to `current_user_led_crew_ids()` / approval set. Every lead gate = `coalesce(...,false)` + `IS NULL` first-line raise + `lead_worker_id IS NOT NULL` in the helper (spec-131 coalesce trap on a NEW write surface, H-NULLGATE).

## 4. Data model (additive-first)

**New tables:** `crews` (project, name, `lead_worker_id` nullable-but-gate-requires, `operator_user_id` set, `default_day_rate` money/ungranted, `kind` dc|subcon, active; `UNIQUE(project,name) WHERE active`); `crew_members` (`UNIQUE(worker_id) WHERE removed_at IS NULL` — one active crew per human; tombstone not delete); `crew_work_packages` (crew↔WP edge; frozen after first labor_log; append-only tombstone); `crew_registrations` (staging, **decoupled from `auth.uid()`** — the phoneless structural fix; `onboarded_by_worker`, `witnessed_by`, status `pending→approved|rejected`); `crew_member_moves` (append-only history).
**New columns:** `workers.crew_id` (current-pointer, granted), `.cost_confirmed_at` (granted — drives the รอยืนยัน chip) / `.cost_confirmed_by` (ungranted), `.deactivated_at` / `.deactivation_reason` / `.do_not_rehire`, **`.national_id`** (+ firm-wide partial-unique index incl. inactive + mod-11 CHECK), `worker_project_moves.crew_id`.
**New enum value:** `contractor_consent_kind += 'processing_basis_employment'` (lawful-basis, lead-attested — kept distinct from subject-given `'pdpa_data'`).
**New RPCs (all SECURITY DEFINER):** `create_crew` / `reassign_crew_lead` (STAFF_APPROVAL_ROLES); `crew_lead_add_member` (own-crew/operator; **inlines** insert, NO money params, national-ID+age gate, collision→raise); `crew_lead_edit_member` (name/phone/emergency only); `assign_crew_to_wp`/`remove_crew_from_wp` (own-crew, frozen post-first-log); `approve_crew_registration(p_id,p_pay_type,p_day_rate,p_employment_type)` / `reject_crew_registration` (approval set); `confirm_worker_cost(p_worker,p_level)` (super_admin, stamps `cost_confirmed_at`); `record_worker_consent_proxy` (own-crew, `p_document_id` NOT NULL, witness); `get_my_crew_members()` / `get_crew_registration_queue()`; `current_user_led_crew_ids()` (helper, `lead_worker_id IS NOT NULL` guarded).
**Reused UNCHANGED:** `create_worker` (bootstrap only), `assign_worker_to_project` + `worker_project_moves`, `create_worker_invite`/`claim_worker_invite`, `set_worker_level`, `set_worker_day_rate`, `employee_id_counters`, the `staff_registration_attachments` supersede pattern (template for id_card capture), `record_worker_consent`/`revoke_contractor_consent`.
**Resolve FIRST (U0):** `is_back_office` EXCLUDES `procurement_manager` while `WORKER_ROSTER_ROLES` + the workers SELECT policy INCLUDE it — pick one SSOT before adding surfaces.

## 5. Consent / PDPA (⚖️ counsel-pending)

One home: `contractor_consents(worker_id)`, never `staff_consents` (auth-user-keyed → unreachable for phoneless). **Legal reframe:** a lead attesting "I got his consent" is void under Thai PDPA s.19 (subject must consent; lead↔member economic dependency = coerced). So the **phoneless lane** stores kind `'processing_basis_employment'` (legitimate-interest / contractual-necessity for **site-access + pay**), requiring a **delivered versioned notice** + **neutral witness** + a mandatory hash-distinct `document_id`. **Phone-owner lane** stores subject-given `'pdpa_data'` at self-claim. Floor accepts either kind. **Purpose-scoped:** activation needs only site-access+pay; ADR-0060 cost use and ADR-0061 ecosystem use each need their OWN granted purpose (realistically only at self-affirm). **Sensitive (s.26):** minimise — ID number + allowed face photo; redact religion on capture; write fails closed unless basis exists. **Age:** <18 flagged, 15–17 guardian artifact, <15 blocked. **Withdrawal:** off-app request to any back-office actor (not the lead); separates operational-deactivate from legal-withdraw; retention-vs-erasure = counsel decision. **Migrant/subcon:** `crew_lead_add_member` REFUSES a `kind='subcon'` crew — subcon members stay in `subcontract_crew_members` (spec 258), basis is the subcon's DPA duty.

## 6. Surfaces / routes

- **Crew-lead surface:** `/projects/[projectId]/crew` ("ทีมของฉัน") — a tab on the project cockpit (where site staff live). Server Component; resolves led/operated crews via the worker-predicate (role-home MUST route a crew-leading `contractor` here — Unit 4). **Bulk fast-add** (name + national-ID; id_card/notice deferred). **Shows the lead his crew's WP cost = sell/560** (D4); pay/markup hidden; **suppressed under SA-proxy**.
- **PM/PD confirmation queue:** `/registrations/crew` — reuse the `/registrations` shape; approval-set + admin-client (legit baht: sets day_rate/pay_type/employment_type; super sets level). Shows lead-captured facts, id_card preview, same-name / same-national-ID cluster warnings, an explicit **identity attestation** checkbox (H-EVIDENCE).
- **278 muster hook:** reads `crew_members` (active, `cost_confirmed_at IS NOT NULL`) UNION `crew_id IS NULL` workers on the project (unled pool + supervisors visible — H-UNLED). Pending = markable-present but NOT labor-loggable. Muster must be **offline-first** (site has 1–2 bars); id_card = capture-now-upload-later.
